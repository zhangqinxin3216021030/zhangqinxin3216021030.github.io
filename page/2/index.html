HTTP报文

HTTP的报文有很多种，其中有请求报文、响应报文、通用报文
实体报文。
首先我们来讲一讲请求报文，请求报文由请求行、请求头部、
空行、请求数据4个部分组成。
1、请求头，请求行由请求方法字段、URL字段和HTTP协议版本
3个字段构成。常见的请求方法在上一篇博客中已经有写到。
2、请求头部，请求头部由关键字/值对组成，每行一对，关键字和值用英文冒号“:”分隔。
请求头部通知服务器有关于客户端请求的信息，典型的请求头有：
User-Agent：产生请求的浏览器类型。
Accept：客户端可识别的内容类型列表。
Host：请求的主机名，允许多个域名同处一个IP地址，即虚拟主机。
3、空行，最后一个请求头之后是一个空行，发送回车符和换行符，通知服务器以下不再有请求头。
4、请求数据，请求数据不在GET方法中使用，而是在POST方法中使用。POST方法适用于需要客
户填写表单的场合。与请求数据相关的最常使用的请求头是Content-Type和Content-Length。
—–HTTP响应报文：
HTTP响应报文也由三部分组成，分别是：状态行、消息报头、响应正文。
状态行格式如下：
HTTP1.1 200 ok ，其中HTTP1.1表示服务器HTTP协议的版本；
200表示服务器发回的响应状态代码；ok表示状态代码的文本描述。
—–HTTP通用报文：
Connection：用于表示连接是否可持续。
Cache-control：用于控制缓存信息。Cache-control：no-cache//防止缓存过期的网页信息，要求
获取最新缓存。
cache-control：no-cache，max-age=0，must-revalidate,no-store
//no-cache,表示不缓存过期信息。
//no-store，不要进行缓存。
Transfer-Encoding：
表示输出的内容长度不能确定，需要分块处理。对于动态的内容或者在发送数据前不能判定长度的情况下，
可以使用分块的方法来传送编码；对于静态的内容或者发送数据可以预判长度的情况下，可以使用 content-length 来标识。
Upgrade：升级为其他协议。
Via：代理服务器相关信息。
Pragma：报文指令。如pragma：no-cache
Trailer：报文末端的首部一览。
—–实体报文：
实体报文是HTTP报文的一部分，如果把HTTP报文看作因特网货运系统中的箱子，那么实体报文就是报文中实际的货物。
内容编码：服务器在接受数据后会对其进行编码，并将编码的数据储存在实体报文中，并发送给对方。
2、传输编码：内容编码是对报文的主体进行的可逆变换，是和内容的具体格式细节紧密相关的。
传输编码也是作用在实体主体上的可逆变换，但使用它们是由于架构方面的原因，同内容的格式无关。使用传输编码
是为了改变报文中的数据在网络上传输的方式。
经过内容编码的报文，只是对实体部分进行了编码。而对于传输报文来说，编码的作用作用与整个报文上，报文自身的
结构发送了改变。
断点传输： Range用于断点的传输使用到条件请求，来查看服务器是否修改资源，资源是否过期；服务器资源是否在两次
下载之间发生改变，如果发生了变化。通过412状态码知道发生变换，得重新获取响应。如果没发送变化可从断点出重新开始。

HTTP返回状态

HTTP返回状态有 1xx 表示信息正在处理
2xx 表示正常处理完毕，如200 ok，204no Content,206 Partial Content
3xx 重定向，表示要完成请求必须更进一步操作，如301 Move Permanently，302 Found
4xx 客户端错误，如400Bad Request ，403 Forbiden， 404 not Found
5xx 服务器错误，服务器未能实现合法的请求
解析：200 ok只是服务器接收到了信息而已，就算密码错误等等操作，服务器也会返回200 ok
204no Content，表示：无实体部分的返回。
206 Partial Content 表示：只返回了部分内容。
301 Move Permanently 表示你的请求的资源信息已经永久变更了。客户端会把新的地址信息返回给你。
400Bad Request 表示：你的请求出现语法错误。
403 Forbiden 表示：你的请求访问被服务器拒绝了。
404 not Found 表示：你输入了错误的资源地址。

http协议

http协议是网络客户端与服务器之间传输的基础，http由客户端
服务端两个部分组成，其中客户端发送请求给服务端，服务端返回
响应给客户端，其中数据是根据URL进行定位的，URL的结构有协议
主机：Host，路径：path 查询参数：Query，服务端接收到URL会
进行解析并返回相应的数据。HTTP的请求有多种如 GET 获取资源
POST:创建资源 put：更新资源 DELETE：删除资源。服务端在接受
请求后会返回响应状态码其中特别的几个是400–499为客户端错误
就是众所周知的404 not fount 500–599 为服务端错误。http协议是
无状态的，如果需要记录状态则需要使用cookie和session保持对话。
get的安全型不如post，传送的数据量get小于post，options请求可以
查询服务端可以使用的请求方法

字符串

字符串顾名思义是由字符组成，但其特殊之处在于其是以\0
结尾的一串数字，0标准着字符串的结束，但它不是字符串。同
时字符串是以字符数组的形式存在，同时以数组或指针的形式访问。
我们可以这样定义一个字符串 char word[]=”Hellow”;
也可以 char line[10] =”Hellow”
这里line后面的中括号里的10代表10个字节的意思，后面的字符串
不能超过这个字节数。同时字符串还有很多与之对应的函数如strlen,
其作用是返回字符串的长度；strcmp，其作用是比较两个字符串。
字符串还有很多运用等着我们去挖掘，我们下期再见。

指针及其用法 

 指针作为c语言的重点，其作用是十分巨大的。其最主要的作用

是可以指向对应变量的地址，进而通过地址对该变量进行修改。其
最基本是书写形式是这样的 int *p=&i；
数组与指针有着秒不可言的关系。可以说数组是一种特殊的指针。
指针在指向数组时不用&，但指向数组对应的某个值时要&。在遍历
数组时可以用指针进行遍历。同时指针间也可以进行加减运算。这就
是指针辣。
